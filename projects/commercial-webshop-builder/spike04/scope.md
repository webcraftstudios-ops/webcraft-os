# Spike04 Scope

## Core question

Can a sufficiently specified request produce a useful sourcing result through a bounded manual process?

A useful result may be:

- a plausible product lead;
- a strong specialist or supplier route;
- a useful community route;
- an improved specification;
- a defensible non-match;
- a clear request for more information.

## In scope

- manual sourcing for a small number of cases;
- missing-information checks;
- channel selection;
- search-term creation;
- multilingual terminology where useful;
- specification refinement;
- objective lead scoring;
- active-time measurement;
- documented next action;
- preparation of a final Spike04 decision.

## Out of scope

- purchase, payment or commission;
- automatic collection or matching;
- customer accounts;
- public marketplace or storefront;
- guarantee that a part will be found;
- production application work.

## Completion criteria

Spike04 is complete only when:

1. multiple sourcing-ready cases have been assessed;
2. the manual process is repeatable;
3. useful outcomes or defensible non-matches are documented;
4. time per case is known;
5. intake blockers are identified;
6. a final decision is recorded.

## Current result

The phase is only partially validated and remains open.
