# GitHub Operating Model — Commercial Webshop Builder

## Rol van GitHub

GitHub is in dit project geen excuus om te vroeg te coderen.

GitHub dient als:

1. projectgeheugen;
2. onderzoeksarchief;
3. beslissingslogboek;
4. MVP-specificatie;
5. validatielogboek;
6. later eventueel codebase voor storefront of automatisering.

## Fasen

### Fase 1 — Research

Bestanden:

- `deep-research-brief.md`
- `tooling.md`
- `decisions.md`

Output:

- onderbouwde richting;
- doelgroep;
- eerste aanbod;
- risico's;
- gratis-first stackkandidaten.

### Fase 2 — MVP

Bestanden:

- `mvp-plan.md`
- `validation-log.md`

Output:

- minimale testversie;
- testkanaal;
- meetpunten;
- succes- en stopcriteria.

### Fase 3 — Build

Pas starten wanneer validatie voldoende sterk is.

Mogelijke toekomstige mappen:

- `storefront/`
- `automation/`
- `analytics/`

## Commitregels

Gebruik kleine, duidelijke commits.

Voorbeelden:

- `Add initial research brief`
- `Update MVP hypothesis`
- `Log validation experiment 001`
- `Document tooling decision`

## Scopebewaking

Niet mengen met andere Webcraft OS-code tenzij het expliciet over runtime, agents of automatisering gaat.

Alles voor dit project blijft onder:

`projects/commercial-webshop-builder/`

## Beslissingsregel

Geen code bouwen zolang `mvp-plan.md` geen scherpe hypothese en `validation-log.md` geen testopzet bevat.
