# Run 002 Selection Criteria — Spike04

## Doel

Run 002 mag niet zomaar een nieuwe willekeurige zoekopdracht zijn. De tweede test moet bewijzen of betere intake of bevestigde specialistinformatie de kans op een bruikbare lead verhoogt.

## Run 002 mag starten wanneer

Minstens één van deze voorwaarden klopt:

```text
A. Alfaholics of Nardi-Personal bevestigt nieuwe specs voor 7Xo88NP
B. Er komt een nieuwe case binnen met foto, partnummer of duidelijke variant
C. De intake krijgt tijdelijk een extra detailblok en levert een beter gespecificeerde case op
```

## Goede Run 002-case

Een goede case bevat minimaal:

```text
merk
model
bouwjaar/periode
onderdeel
origineel/repro/used voorkeur
foto of partnummer indien beschikbaar
variant_details indien relevant
compatibility_uncertain ja/nee
```

## Voor stuurwielcases extra

```text
stuurwielmerk
stuurwieldiameter
stuurwiellijn/model
hub/boss/adaptor bekend ja/nee/onbekend
bolt pattern / PCD indien bekend
airbag/non-airbag context
```

## Slechte Run 002-case

Niet gebruiken als Run 002:

```text
onderdeel = test
onderdeel = algemeen interieurdeel
geen model/bouwjaar
geen foto/variant/partnummer bij compatibiliteitsgevoelig onderdeel
```

## Vergelijking met Run 001

Run 002 moet kunnen meten:

```text
Was de aanvraag concreter?
Was er minder refinement nodig?
Vond de sourcing sneller een source lead?
Kwam er wel een productlead?
Welke intakevelden maakten verschil?
```

## Succescriterium Run 002

```text
lead_score >= 4
of
minder dan 60 minuten nodig om een duidelijke source/productroute te bepalen
of
bewijs dat betere intake de zoekrichting sterk verbetert
```

## Geen Spike05 tenzij

Spike05 mag pas starten als minimaal één van deze twee waar is:

```text
1. twee cases tonen herhaalbaar proces en duidelijke source/productroute
2. één case levert een productlead en één andere case bevestigt dat intake/refinement herhaalbaar is
```
